# Inventory Optimisation Agent — Python dependencies.

# late 2024 / early 2025; the floors below all do.
#
# Tested with:   Python 3.11, 3.12, 3.13
# Recommended:   Python 3.12 (best wheel coverage for all libs)

# HTTP client (async, used for Maximo OSLC calls)

# JWT — python-jose with cryptography backend

# Settings via .env
pydantic-settings>=2.5.0

# Pydantic v2 (pulled by fastapi, floor pinned to a 3.13-friendly release)

# ─── Persistence (Q1.1) ───────────────────────────────────────────────────────

# numpy 2.1+ and scipy 1.14+ are the first releases with Python 3.13 wheels.
numpy>=2.1.0
scipy>=1.14.0

# statsforecast pulls numba+llvmlite, which lag a release behind Python.
# Skip the install entirely on Python 3.13; the forecasting service detects
# the missing import and falls back to the NumPy bootstrap quantile path.
# Quality-wise the bootstrap is good enough for an initial pilot — re-enable
# statsforecast on Python 3.12 (or once numba ships 3.13 wheels you can use).
statsforecast>=1.7.6 ; python_version < "3.13"

# ─── Scheduler (Q1.2) ─────────────────────────────────────────────────────────
